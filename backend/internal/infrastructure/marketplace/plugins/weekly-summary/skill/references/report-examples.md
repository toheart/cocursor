## Best Practices

### Accomplishment Description Guidelines

**Good descriptions:**
- "Completed JWT authentication API with login/logout/token refresh"
- "Resolved large directory monitoring lag, reduced CPU usage by 40%"
- "Refactored 4-step config wizard, added validation"

**Poor descriptions:**
- "Wrote some code"
- "Fixed a few bugs"
- "Did authentication-related work"

### Data Presentation

- Use specific numbers: "+520 lines" not "a lot of code"
- Show comparison: "reduced by 40%" not "optimized performance"
- Be explicit about scope: "38 files changed" not "changed many files"
